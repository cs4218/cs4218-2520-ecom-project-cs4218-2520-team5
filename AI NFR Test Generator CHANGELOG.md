## 2026-04-12 — Fix coverage run tracking: scope URL, add polling loop, remove race condition

- **"Wait for New Coverage"**: reduced from 120s to 15s (just enough for GitHub to register the dispatched run before we look it up)
- **"Get New Workflow Runs"**: scoped URL from `/actions/runs` (all workflows) to `/actions/workflows/coverage.yml/runs?branch={branch}` — previously picked the wrong run (e.g., `main.yml`) which has no `coverage-summary` artifact
- **"Select New Latest Run"**: removed `status === "completed"` filter — now picks the most recently *created* run regardless of status, so we get the run we just dispatched
- **"Get New Artifacts"**: URL now references `$('Select New Latest Run').item.json.*` directly instead of `$json.*`, so it stays correct regardless of what the preceding node in the loop passes through
- **New node "Poll Coverage Run Status"**: `GET /actions/runs/{run_id}` to check the status of the specific run we identified
- **New node "Check Coverage Completed"**: IF node — `$json.status === "completed"`; true branch proceeds to `Get New Artifacts`, false branch waits and retries
- **New node "Wait to Poll Again"**: 30s wait on the false branch, loops back to `Poll Coverage Run Status`

## 2026-04-12 — Fix repo field paths, add author header enforcement, fix Unicode literals (Lynette)

- **Get Repo Info node**: Fixed `defaultBranch` expression from `$json.default_branch` → `$json.source.default_branch` and `pushedAt` from `$json.pushed_at` → `$json.parent.pushed_at` to correctly traverse forked repo API response shape
- **Format Test File node**: Added step to prepend an author comment (`// Auto-generated by n8n AI Testing Workflow...`) to every generated test file if one is not already present; renumbered steps accordingly and removed AI branch (`aiBranch`) from output
- **Multiple nodes**: Replaced Unicode escape sequences (`\u2190`, `\u2014`, `\u2192`) with literal characters (`←`, `—`, `→`) for readability in node comments/content
